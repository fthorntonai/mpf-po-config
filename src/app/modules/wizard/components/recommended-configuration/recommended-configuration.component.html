Download PDF A published offer with public plan and the plan must have a &paymentFrequency pricing.
  • Model this deal with a single private offers, create a private offer on the same public plan (@term term). Note, this example assumes customer will subscribe in @thisMonth @thisYear.   Private Offer #1 for the first year
    • Start date: @thisMonth + "’" + thisYear or Accepted date
    • End date: @thisMonth + "’" + (@thisYear + (@term * @numberOfPayments)
    • Billing term
    • Price per payment: @totalAmount / @numberOfPayments
  • Customer will need to:
    • Accept the private offers to lock the price for the @numberOfPayments + @term.
  • Subscribe (purchase) the product once using the @term term.
  • Customer will be charged $@contractTotal
  • At the end of the @numberOfPayments + @term, ISV will need to set up a new private offer on the same public plan or customer will fall back to the list price. Alternatively, Customer should switch off auto-renew if they no longer want the product after the @numberOfPayments + @term.
